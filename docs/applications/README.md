# Applications

Applications are documented by function rather than alphabetically so the setup
reads like a workstation design, not a package dump.

Each category gets its own folder with a `README.md` landing page, and each
individual tool should get its own file inside that folder.

Examples of the intended shape:

- `applications/terminal/<tool>.md`
- `applications/editors/<tool>.md`
- `applications/productivity/<tool>.md`

Each category page should answer:

- which apps belong in the category
- why they are part of the default setup
- how they are installed or restored
